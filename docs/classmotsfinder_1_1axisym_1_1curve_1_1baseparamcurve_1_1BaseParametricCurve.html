Detailed Description

Base class for parametric curves in the x-z-plane.

This base class does the management of component functions (freezing/unfreezing/resampling/...). Interpreting the component functions and transforming to Cartesian coordinates is the responsibility of child classes. These have to implement:

  • from_curve()
  • __call__()
  • _diff()

Return evaluator(s) of any internal numeric expressions.

Subclasses may have multiple expressions representing the curve. Since it is inefficient to repeatedly create evaluators when the underlying expressions don't change, the BaseCurve class can handle freezing (i.e. reusing) evaluators. Any evaluators returned from this function will participate in this mechanism.

Parameterize the curve by its arc length.

After calling this method, all tangent vectors will have (roughly) equal lengths. The parameter will be scaled such that the domain remains [0,pi]. An optional metric can be given to parameterize the curve by its length in curved space.

Parameters
metricOptional metric for computing tangent norms. If not given, the curve will have constant "speed" in coordinate space. If given, it will have constant "speed" in the curved space described by the metric.
coord_spaceWhether to reparameterize in internal coordinate space. Default is False.
rtolRelative tolerance for solving the initial value problem (IVP), see below. Default is 1e-12.
atolAbsolute tolerance for solving the IVP. Default is 1e-14. Both rtol and atol are passed to scipy.integrate.solve_ivp and control the adaptive step size such that the local error estimates are less than atol + rtol * abs(s), where s is the reparameterization function (see below).
numOptional new resolution for the x and z component functions. The default is to keep the current resolution.
blendHow much to consider metric versus flat coordinate space. Default is 1.0, i.e. use provided metric.

Notes

Consider the reparameterized curve

\[ \tilde\gamma(\lambda) := \gamma(s(\lambda)), \]

where \(\gamma\) is the current curve. The reparameterization is performed by numerically integrating the differential equation we get by demanding that

\[ \Vert\tilde\gamma'(\lambda)\Vert = \frac{\ell(\gamma)}{\pi}, \]

where \(\ell(\gamma)\) is the length of the curve. As a result, the domain remains [0,pi] as before. We hence get the following nonlinear ODE for \(s\)

\[ s'(\lambda)=\frac{\ell(\gamma)/\pi}{\Vert\gamma'(s(\lambda))\Vert} =: f(s(\lambda)). \]

Starting with \(s(0) = 0\), this equation is integrated numerically using a Runge-Kutta [1] integration method provided by scipy.integrate.solve_ivp. Afterwards, the x and z component functions are re-computed to match the new parameterization.

In practice, we do not need to compute \(\ell(\gamma)\) and instead integrate the above ODE with an arbitrary constant (1.0) until we have s == pi.

References

[1] Press, William H., et al. Numerical recipes 3rd edition: The art of scientific computing. Cambridge university press, 2007.

Experimental reparameterization focussing on parts with high curvature.

This is similar to reparameterize(), but instead of requiring

\[ \Vert\tilde\gamma'(\lambda)\Vert_g = \mathrm{const}, \]

we implement an experimental formula which slows the parameterization at places of high curvature in coordinate space. Specifically,

\[ \Vert\tilde\gamma'(\lambda)\Vert_g = c\ \exp\left(-\alpha\,\kappa^\beta\right), \]

where alpha and beta are constants and \(\kappa(\lambda)\) is the curvature of the curve in coordinate space. The constant c is chosen such that the reparameterized curve keeps its domain [0,pi].

As in reparameterize(), the norm on the L.H.S. above is the flat-space norm by default, but may be chosen to be any norm by supplying a metric to this function.

Parameters
metricOptional metric for computing tangent norms (see above).
coord_spaceWhether to reparameterize in internal coordinate space. Default is False.
alphaParameter \(\alpha\) for the above formula. Default is 0.1.
betaParameter \(\beta\) for the above formula. Default is 0.5.
rtolRelative tolerance for solving the initial value problem (IVP), see below. Default is 1e-12.
atolAbsolute tolerance for solving the IVP. Default is 1e-14. Both rtol and atol are passed to scipy.integrate.solve_ivp and control the adaptive step size such that the local error estimates are less than atol + rtol * abs(s), where s is the reparameterization function (see reparameterize()).
numOptional new resolution for the x and z component functions. The default is to keep the current resolution.
blendHow much to consider metric versus flat coordinate space. Default is 1.0, i.e. use provided metric.

Reparameterize the curve using the square of its extrinsic curvature.

The steps of this strategy are:

  • curve is parameterized by its arc length (and possibly upsampled in this step to keep as much of its shape as possible)
  • the square \(k^{AB}k_{AB}\) of the extrinsic curvature of the curve is sampled as a function and represented by a cosine series
  • this series is "smoothed" by damping the coefficients of the series representation exponentially
  • the resulting (smoothed) function is taken as a "speed function" for the reparameterization, which also resamples the curve back to its original (or desired target) resolution num

By starting with arc length parameterization, we ensure that repeated reparameterizations are well behaved (except for eventual precision loss by multiple changes of representation, of course).

Parameters
metricOptional metric to use for the arc length parameterization and also for computing the extrinsic curvature. Since the curves are represented and manipulated in coordinate space, however, it is recommended to not use the spacetime but the flat (default) metric here.
coord_spaceWhether to reparameterize in internal coordinate space. Default is False.
rtolRelative tolerance for solving the initial value problem (IVP), see below. Default is 1e-12.
atolAbsolute tolerance for solving the IVP. Default is 1e-14. Both rtol and atol are passed to scipy.integrate.solve_ivp and control the adaptive step size such that the local error estimates are less than atol + rtol * abs(s), where s is the reparameterization function (see reparameterize()).
numOptional new resolution for the x and z component functions. The default is to keep the current resolution.
res_factorFactor by which to increase the resolution for the initial arc length parameterization. Default is 1.0.
max_resResolution limit when applying res_factor. Default is 6000.
smoothingSmoothing factor to apply. A value of 0 skips any smoothing. Note that due to the finite sampling resolution, the resulting speed function may become negative (i.e. reparameterization will fail). Smoothing will typically help in these cases. Default is 0.05.
exponentTaking the square of the extrinsic curvature results in high density variations. These can be further reduced by taking a fractional power of \(k^{AB}k_{AB}\) as the speed function. Default is 0.5, i.e. taking the square root.
