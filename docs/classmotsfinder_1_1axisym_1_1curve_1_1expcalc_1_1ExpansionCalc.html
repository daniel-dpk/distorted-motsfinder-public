Detailed Description

Abstract base class for computing the expansion at one point.

This class serves as coordinator for computing the expansion and functional derivatives w.r.t. the horizon function. Sub classes need only implement a small number of computational methods.

The purpose of having a separate class hierarchy for computing the expansion (as opposed to doing all the computations inside the curve classes) is to be able to store a number of interim results valid only for the results at one point of the surface. Including these as cache in the curve classes would in principle be possible. To ease management of cache invalidation (when computing at a different point), the complete cache should live on one object. The ExpansionCalc class and its sub classes can be interpreted as such a cache, with added functionality to do the necessary computations using the cached values.

Compute analytical functional derivatives of the expansion.

This may raise a NotImplementedError, indicating that FD differentiation needs to be performed.

Parameters
hdiffDerivative order of h to differentiate the expansion by (see below). E.g., a value of 0 will compute \(\partial_h \Theta\).

Notes

In general, due to the ultra-local dependency of the expansion on h and its first two derivatives, we can treat the variational differentiation like a simple partial differentiation. This can also be seen by taking the definition

\[ (\delta\Theta)(h)\Delta := \frac{d}{d\varepsilon}\Big|_{\varepsilon=0} \Theta(h+\varepsilon\Delta) \]

and separating the terms based on the derivative order of \(\Delta\). The result will be of the form

\[ (\delta\Theta)(h)\Delta = \partial_h\Theta \Delta + \partial_{h'}\Theta \Delta' + \partial_{h''}\Theta \Delta''. \]

These three terms are computed here using

\[ \partial_f \Theta = \frac{A_f}{D^{3/2}} - \frac{3}{2} \frac{A D_f}{D^{5/2}} + \frac{B_f}{D^{1/2}} - \frac{1}{2} \frac{B D_f}{D^{3/2}} + \frac{C_f}{D} - \frac{C D_f}{D^2} - \partial_f \,\mathrm{tr} K, \]

where f is one of h, h', h''.

The terms A, B, C, and D are defined in [1], but here we repeat them for convenience:

\begin{eqnarray*} A &:=& -s^i s^j \partial_i s_j - \frac{1}{2} s^i (\partial_i g^{kl}) s_k s_l \\ B &:=& (\partial_i g^{ij}) s_j + g^{ij} \partial_i s_j + (\partial_i \ln\sqrt{g}) s^i \\ C &:=& K^{ij} s_i s_j \\ D &:=& s_i s^i. \end{eqnarray*}

References

[1] Thornburg, Jonathan. "A fast apparent horizon finder for three-dimensional Cartesian grids in numerical relativity." Classical and quantum gravity 21.2 (2003): 743.

Compute derivatives of the expansion using finite differencing.

Since the expansion depends on h and its derivatives only ultra-locally, a reasonable approximation to the variational derivative of the expansion w.r.t. h can be obtained by varying h (or derivatives) point-wise, i.e. compute the usual partial derivative of the expansion w.r.t. h. This can be approximated using a finite difference differentiation, which is done in this function. Note that irrespective of the accuracy of this approximation, the test whether the expansion has the desired value (e.g. 0.0 for a MOTS) is independent of the results computed here.

Compute (derivatives of) the normalized covariant normal.

Parameters
diffDerivative order to compute. Default is 0.
Returns
NumPy ndarray with diff+1 axes and indices i1,i2,...,k corresponding to \(\partial_{i_1}\partial_{i_2}\ldots\nu_k\). For example, for diff==0, returns the three components of nu.

Notes

Given the non-normalized components \(s_i\) of the covariant outward pointing normal on the surface, we compute

\[ \nu_i = \frac{s_i}{\sqrt{D}}, \qquad D := g^{kl} s_k s_l. \]

From this formula, we get the x,y,z derivatives

\[ \partial_i\nu_j = \frac{\partial_i s_j}{\sqrt{D}} - \frac{s_j}{2 D^{3/2}} D_i \]

and

\[ \partial_i\partial_j\nu_k = \frac{\partial_i \partial_j s_k}{\sqrt{D}} - \frac{1}{2 D^{3/2}} \Big( (\partial_j s_k) D_i + (\partial_i s_k) D_j + s_k D_{ij} \Big) + \frac{3}{4} \frac{s_k}{D^{5/2}} D_i D_j, \]

where

\begin{eqnarray*} D_i &:=& \partial_i D = (\partial_i g^{kl}) s_k s_l + 2 g^{kl} s_k\,\partial_i s_l \\ D_{ij} &:=& \partial_i\partial_j D \\ &=& (\partial_i \partial_j g^{kl}) s_k s_l + 2 (\partial_i g^{kl}) s_k\,\partial_j s_l + 2 (\partial_j g^{kl}) s_k\,\partial_i s_l \\&& + 2 g^{kl} \big( (\partial_j s_k)(\partial_i s_l) + s_k \partial_i \partial_j s_l \big). \end{eqnarray*}

Compute derivatives of x,y,z w.r.t. lambda and phi.

This computes the derivatives of the Cartesian coordinates x,y,z w.r.t. the surface intrinsic coordinates lambda and phi based on the usual transform rules

\begin{eqnarray*} x = \rho(\lambda)\cos\varphi,\quad y = \rho(\lambda)\sin\varphi,\quad z = z(\lambda), \end{eqnarray*}

where \(\rho\) is the x-component of the curve and z its z-component. The results are evaluated at \(\varphi = 0\).

Returns
For diff==1, return the first derivatives with indices dx[A,i] meaning \(\partial_A x^i\), where we have \((x^i) := (x,y,z)\) and \((\partial_A) := (\partial_\lambda, \partial_\varphi)\). For diff==2, second derivatives are returned with indices ddx[A,B,i] meaning \(\partial_A\partial_B x^i\). The same pattern holds for diff==3. If diff==None, a list [dx, ddx, dddx] is returned.
Parameters
diffDerivative order. One of 1, 2, 3. Default is 1. If explicitely set to None, all three implemented orders are returned.

def extrinsic_curvature (   self,
  trace = False,

)

Compute the extrinsic curvature.

Parameters
traceIf True, returns the trace of the extrinsic curvature. Default is False. May not be used together with square.
squareIf True, returns the square \(k_{AB} k^{AB}\). Default is False. May not be used together with trace.
Returns
If trace=square=False, a NumPy 2x2 array containing the components of k_AB. Otherwise, returns a float.

Notes

To get the components \(k_{AB} = -\nabla_A \nu_B\), first note that k annihilates any components transverse to the surface \(\sigma\) (see e.g. [1]), i.e. for any point \(p \in \sigma\)

\[ k(v_p, X_p) = 0 \qquad \forall\,X_p\in T_p M, \]

where \(v\) is any vector field normal to \(\sigma\), for example the normal \(\nu\) in the current slice \(\Sigma\) or the future pointing normal n of the slice in spacetime. Hence, we will in the following restrict all objects to \(\sigma\). For example,

\[ dx^\mu\big|_\sigma = \frac{\partial x^\mu}{\partial u^A}\ du^A =: x^\mu_{,A}\ du^A, \]

where \(u^A = \lambda,\varphi\). The \(x^\mu_{,A}\) are computed in diff_xyz_wrt_laph(). Note that \(x^0_{,A} = 0\) since \(x^0 = t\) does not depend on \(\lambda\) or \(\varphi\). Observing further that \(\partial_A = x^a_{,A}\partial_a\), we get

\begin{eqnarray*} \nabla_{\!\partial_A} \nu_B &=& \big[x^a_{,A} \nabla_a \underline\nu\big]_B \\ &=& x^a_{,A} \big[ (\partial_a\nu_\beta - \Gamma^\alpha_{a\beta}\nu_\alpha)\ dx^\beta \big]_B \\ &=& x^a_{,A} \big[ (\partial_a\nu_b - \Gamma^c_{ab}\nu_c)\ dx^b \big]_B \\ &=& x^a_{,A} (\partial_a\nu_b - \Gamma^c_{ab}\nu_c) x^b_{,B} \\ &=& x^a_{,A} x^b_{,B} (\partial_a\nu_b - {}^{(3)}\Gamma^c_{ab}\nu_c). \end{eqnarray*}

The third equality is due to \(dx^0\big|_\sigma = 0\) and \(\nu_0 = 0\). The reason we can take the Christoffel symbols of the 3-metric in the slice is that, by their definition and using \(g_{ab} = {}^{(4)}g_{ab} + n_a n_b\),

\begin{eqnarray*} (\Gamma^k_{ab} - {}^{(3)}\Gamma^k_{ab}) \nu_k &=& \frac{1}{2} (g^{kc} - n^k n^c) \big[ - \partial_c (g_{ab} - n_a n_b) + \partial_a (g_{bc} - n_b n_c) + \partial_b (g_{ca} - n_c n_a) \big] \nu_k \\&& - \frac{1}{2} g^{kc} \big[ - \partial_c g_{ab} + \partial_a g_{bc} + \partial_b g_{ca} \big] \nu_k \\ &=& \frac{1}{2} \nu^c \big[ \partial_c (n_a n_b) - \partial_a (n_b n_c) - \partial_b (n_c n_a) \big] \\ &=& 0. \end{eqnarray*}

The first equality is due to \(n^k \nu_k = 0\) (n is orthogonal to any horizontal vectors, i.e. in \(T_p\Sigma\)) and the last equation due to \(n_\mu = 0\) for \(\mu \neq 0\).

References

[1] D. Giulini. "Dynamical and Hamiltonian Formulation of General Relativity". In: Springer handbook of spacetime. Ed. by A. Ashtekar and V. Petkov. Springer, 2014. Chap. 17.

◆ get_dh_ABCDtrK()

def get_dh_ABCDtrK (   self)

Compute the derivative of A, B, C, D, tr(K) w.r.t. h.

May raise NotImplementedError to indicate numerical differentiation should be done.

Refer to the definition of A,B,C,D in the documentation of _diff(). The terms computed here are:

\[ \partial_h A = -2(\partial_h s^i) s^j \partial_i s_j - s^i s^j \partial_h \partial_i s_j - \frac{1}{2} (\partial_h s^i) (\partial_i g^{kl}) s_k s_l - \frac{1}{2} s^i (\partial_h \partial_i g^{kl}) s_k s_l - s^i (\partial_i g^{kl}) s_k \partial_h s_l \]

\[ \partial_h B = (\partial_h \partial_i g^{ij}) s_j + (\partial_i g^{ij}) \partial_h s_j + (\partial_h g^{ij}) \partial_i s_j + g^{ij} \partial_h \partial_i s_j + (\partial_h \partial_i \ln\sqrt{g}) s^i + (\partial_i \ln\sqrt{g}) \partial_h s^i \]

\[ \partial_h C = \big[(\partial_h g^{ik}) g^{jl} + g^{ik}(\partial_h g^{jl})\big] K_{kl} s_i s_j + g^{ik} g^{jl} (\partial_h K_{kl}) s_i s_j + 2 g^{ik} g^{jl} K_{kl} s_i \partial_h s_j \]

\[ \partial_h D = (\partial_h g^{ij}) s_i s_j + 2 g^{ij} s_i \partial_h s_j \]

\[ \partial_h \mathrm{tr}K = (\partial_h g^{ij}) K_{ij} + g^{ij} \partial_h K_{ij} \]

The individual terms are computed by simply applying the chain rule. We obtain for any quantity f which depends on the coordinates x,y,z:

\[ \partial_h f = (\partial_i f) (\partial_h\gamma)^i, \]

where \(\gamma\) is the curve along which the computation takes place.

◆ get_dhp_ABCD()

def get_dhp_ABCD (   self)

Compute the derivative of A, B, C, D w.r.t. ‘h’`.

May raise NotImplementedError to indicate numerical differentiation should be done.

This implementation is correct iff

\begin{eqnarray*} \partial_{h'} s_i &=& - X_i\\ \partial_{h'} \partial_i s_j &=& - X_{ij}, \end{eqnarray*}

where \(X_i := \partial_i \lambda\) and \(X_{ij} := \partial_i \partial_j \lambda\).

The terms computed here then become (refer to _diff()):

\begin{eqnarray*} \partial_{h'} A &=& 2 X^i s^j \partial_i s_j + s^i s^j X_{ij} + \frac{1}{2} (\partial_i g^{kl}) (X^i s_k s_l + 2 s^i X_k s_l) \\ \partial_{h'} B &=& -(\partial_i g^{ij}) X_j - g^{ij} X_{ij} - (\partial_i\ln\sqrt{g}) X^i \\ \partial_{h'} C &=& -2 K_{ij} X^i s^j \\ \partial_{h'} D &=& -2 X_i s^i \end{eqnarray*}

This method is agnostic as to how the surfaces are represented as long as the quantities \(s_i\), \(\partial_i s_j\), \(X_i\), and \(X_{ij}\) are available.

◆ get_dhpp_AB()

def get_dhpp_AB (   self)

Compute the derivative of A and B w.r.t. h''.

May raise NotImplementedError to indicate numerical differentiation should be done.

This implementation is correct iff

\begin{eqnarray*} \partial_{h''} s_i &=& 0\\ \partial_{h''} \partial_i s_j &=& - X_i X_j. \end{eqnarray*}

We compute here (see also _diff()):

\begin{eqnarray*} \partial_{h''} A &=& s^i s^j X_i X_j \\ \partial_{h''} B &=& -X^i X_i \\ \partial_{h''} C &=& \partial_{h''} D = 0 \end{eqnarray*}

This method is agnostic as to how the surfaces are represented as long as the quantities \(s_i\), \(\partial_i s_j\), \(X_i\), and \(X_{ij}\) are available.

def induced_metric (   self,

  inverse = False

Compute the induced metric on the surface.

This method computes the components of the induced metric in \((\lambda,\varphi)\) coordinates as well as the components of the inverse (i.e. indices upstairs) and derivatives of these components.

Since this class assumes axisymmetry throughout, this method requires (without loss of generality) that the point at which the metric is to be returned is located at phi=0, i.e. y=0 and x>0.

Parameters
diffDerivative order to compute. Default is 0.
inverseWhether to return the (derivatives of the) inverse of the induced metric. Default is False.
Returns
NumPy array with 2+diff axes, such that the indices [A1,A2,...,B,C] correspond to \(\partial_{A_1}\partial_{A_2}\ldots q_{BC}\) for inverse==False and with upstairs indices for invers==True.

Notes

The induced 2-metric q on the surface \(\sigma\) is formally given by

\[ q = \Pi_\sigma g = g\big|_\sigma - \underline{\nu} \otimes \underline{\nu}, \qquad q_{ab} = g_{ab} - \nu_a \nu_b, \]

where \(\nu\) is the outward pointing normal of \(\sigma\) and \(\underline{\nu} = g(\nu,\,\cdot\,)\). The induced metric can easily be expressed in terms of the components of the 3-metric g by expanding these into the cobasis fields of the coordinates \(\lambda, \varphi\) on the 2-surface (and thereby dropping any transversal components). As a result, we get the simple formula

\[ q_{AB} = g_{ij}\ (\partial_A x^i)\ (\partial_B x^j), \]

where A,B = 1,2 and \((\partial_A) = (\partial_\lambda, \partial_\varphi)\). The derivatives of the Cartesian coordinates x,y,z are computed in diff_xyz_wrt_laph().

From this, we easily get the first and second derivatives by applying the chain and product rule:

\begin{eqnarray*} \partial_A q_{CD} &=& (\partial_A g_{ij}) x_C^i x_D^j + g_{ij} (x_{CA}^i x_D^j + x_C^i x_{DA}^j) \\ \partial_A\partial_B q_{CD} &=& (\partial_A\partial_B g_{ij}) x_C^i x_D^j + (\partial_A g_{ij}) (x_{CB}^i x_D^j + x_C^i x_{DB}^j) + (\partial_B g_{ij}) (x_{CA}^i x_D^j + x_C^i x_{DA}^j) \\&& + g_{ij} (x_{CAB}^i x_D^j + x_{CA}^i x_{DB}^j + x_{CB}^i x_{DA}^j + x_C^i x_{DAB}^j). \end{eqnarray*}

Here, \(x_{A}^i := \partial_A x^i\), etc.

◆ ricci_scalar()

def ricci_scalar (   self)

Compute the Ricci scalar of the surface represented by the curve.

The Ricci scalar of a 2-surface is defined as (see e.g. [1]) \(R = q^{AB}R_{AB}\), where q is the induced metric \(q_{ab} = g_{ab} - \nu_a \nu_b\), \(R_{AB}\) is the Ricci tensor \(R_{AB} = R^C_{\ A\,CB}\) and \(\nu\) the covariant outward unit normal of the surface. Here, \(R^A_{\ B\,CD}\) is the Riemann tensor. Note that A,B run over the coordinates \((\lambda,\varphi)\) on the surface and a,b over x,y,z. See induced_metric() for a bit more details on the induced metric q and the coordinate transformation to get the components \(q_{AB}\) we need here.

It is convenient to compute the Ricci scalar from the purely covariant Riemann tensor \(R_{AB\,CD} = q_{AE}R^E_{\ B\,CD}\) as this is antisymmetric in the first and last two index pairs, i.e. it has only one independent component \(R_{\lambda\varphi\,\lambda\varphi}\) in two dimensions. A short calculation reveals

\[ R = q^{AB}R_{AB} = 2 R_{\lambda\varphi\,\lambda\varphi} (q^{\lambda\lambda}q^{\varphi\varphi} - (q^{\lambda\varphi})^2). \]

def riemann (   self)

Compute the components of the Riemann tensor on the surface.

The Riemann tensor computed here is defined as

\[ R^A_{\ B\,CD} = \partial_C \Gamma^A_{DB} - \partial_D \Gamma^A_{CB} + \Gamma^A_{CE} \Gamma^E_{DB} - \Gamma^A_{DE} \Gamma^E_{CB}, \]

where \(\Gamma^{A}_{BC}\) are the Christoffel symbols of the induced 2-metric q.

Due to the antisymmetry in the last two indices, only two components may potentially be nonzero, namely \(R^\lambda_{\ \varphi\,\lambda_\varphi}\) and \(R^\varphi{\ \varphi\,\lambda_\varphi}\). These two components are returned here.
