Detailed Description

This class represents an individual patch (block) of a field.

This base class does not contain any actual field or field component data, it just represents the meta-like information of the patch, e.g. the bounding box, spatial grid resolution (i.e. the deltas) and physical coordinate of the origin.

Construct a new patch object.

Note that since all classes in this module assume the data to represent axisymmetric fields/tensors, we explicitly check that the xz-plane is contained in this patch. This allows the data to contain additional 3-D data around y=0, as will be the case for data containing ghost points at the domain boundaries. The public attribute yidx will hold the index in y-direction at which the xz-plane's data is found. If no ghost points are included, yidx should be zero.

Parameters
originOrigin of this data patch as obtained from the field component block.
deltas3 by 3 matrix containing the deltas for the x, y, z physical grid translation vectors.
boxBounding box of this patch. Should be a BBox object.

Compute the grid point indices closest to given physical coordinates.

This takes a point in physical coordinates and computes the indices of the matrix element that lies closest. This is done in constant time (i.e. no "search" is performed).

Parameters
point3-D point in physical coordinates.
floatingIf True, the returned indices will be of floating point type, the fractional portion denoting the precise location of the point in "grid coordinate space". This may be useful for interpolating the matrix values between the grid points.

Compute the physical coordinates of a grid point.

The result is a point in coordinate space. This works even if the grid point lies outside the patch.

The parameters i, j, k denote the indices of the patch matrix element belonging to the returned point. If k is not given, the two given indices are interpreted as lying in the xz-plane and the y-index is chosen to be at y=0.

Generator to iterate over all grid points in the patch.

It is more efficient to avoid iterating over the whole grid in Python. Instead, many operations can be performed by utilizing numpy's ability to operate on whole matrices. This method should only be used if efficiency is not of highest concern.

Parameters
xz_planeWhether to only output grid points on the xz-plane (i.e. for y=0). If the data contain ghost points in y-direction, this will pick out only those in the xz-plane. Default is True.
ghostNumber of ghost points to skip at the boundaries. Default is 0.
full_outputIf True, the yielded elements will consist of the matrix indices (i,j,k) and the physical coordinates (x,y,z). Otherwise, only the matrix indices are yielded. Default is False.
