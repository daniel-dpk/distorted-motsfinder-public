Create a parametric curve from two component functions.

Parameters
x_fun(exprs.numexpr.NumericExpression) Component function for the x-component of the curve. For curves starting and ending on the z-axis, a suitable choice is a function expanded into a pure sine series (see exprs.trig.SineSeries).
z_fun(exprs.numexpr.NumericExpression) Component function for the z-component of the curve. For curves starting and ending on the z-axis, a suitable choice is a function expanded into a pure cosine series (see exprs.trig.CosineSeries).
freeze(boolean) Since these curves often will not change during their life time, their evaluators can be frozen upon construction of the curve. This makes subsequent evaluation of the curve, its tangent, and its normal (and derivatives thereof) very efficient.
nameName of this curve. This may be used when printing information about this curve or as label in plots.

Parameterize the curve by its arc length.

After calling this method, all tangent vectors will have (roughly) equal lengths. The parameter will be scaled such that the domain remains [0,pi]. An optional metric can be given to parameterize the curve by its length in curved space.

Parameters
metricOptional metric for computing tangent norms. If not given, the curve will have constant "speed" in coordinate space. If given, it will have constant "speed" in the curved space described by the metric.
rtolRelative tolerance for solving the initial value problem (IVP), see below. Default is 1e-12.
atolAbsolute tolerance for solving the IVP. Default is 1e-14. Both rtol and atol are passed to scipy.integrate.solve_ivp and control the adaptive step size such that the local error estimates are less than atol + rtol * abs(s), where s is the reparameterization function (see below).
numOptional new resolution for the x and z component functions. The default is to keep the current resolution.
blendHow much to consider metric versus flat coordinate space. Default is 1.0, i.e. use provided metric.
