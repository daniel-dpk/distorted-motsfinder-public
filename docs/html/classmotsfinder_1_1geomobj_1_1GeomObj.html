Detailed Description

Parent class for geometric objects.

This class manages the evaluators required for computation. Subclasses are free to use however many evaluators are required for their definition.

Subclasses need to implement the following functions:

  • copy() to create an independent copy of the object
  • _create_evaluators() to create one or more evaluators for any internally stored numeric expressions. This is part of the evaluator caching mechanism.

◆ _create_evaluators()

def _create_evaluators (   self)
private

Return evaluator(s) of any internal numeric expressions.

Subclasses may have multiple expressions representing the object. Since it is inefficient to repeatedly create evaluators when the underlying expressions don't change, this base class can handle freezing (i.e. reusing) evaluators. Any evaluators returned from this function will participate in this mechanism.

def override_evaluator (   self,
  evaluator 
)

Context for temporarily overriding the evaluator(s) used.

This may be used to inject stand-in evaluators to temporarily alter the state of the object. The primary use case is to supply the actual evaluator(s) of the expressions to fix them to their current state. An alternative use case is to modify the object for (e.g.) implementing functional derivatives of operators depending on this object ultralocally.
